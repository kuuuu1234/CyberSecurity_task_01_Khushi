# Part B: Password Security Review

# What makes a strong password?
A strong password should:
- Be at least 12 characters long
- Include uppercase and lowercase letters
- Include numbers and special symbols
- Avoid dictionary words or personal information

# Why should passwords be unique?
Using unique passwords prevents multiple accounts from being compromised if one password is leaked.

# What is Multi-Factor Authentication (MFA)?
MFA is a security method that requires multiple verification steps such as:
- Password
- OTP (One-Time Password)
- Biometric verification

# Risks of Password Reuse
- Multiple account compromise
- Identity theft
- Financial loss

# Password Policy (Best Practices)
1. Use minimum 12 characters
2. Combine letters, numbers, and symbols
3. Avoid personal information
4. Change passwords regularly
5. Enable MFA on all accounts

# 🦠 Part D: Cyber Threat Research

# Phishing
- **Definition:** Fraudulent attempt to steal sensitive data via fake emails or websites  
- **Example:** Fake bank email asking login details  
- **Prevention:**  
  - Verify sender  
  - Avoid clicking unknown links  
  - Check website URLs  

---

# Malware
- **Definition:** Malicious software designed to damage or gain unauthorized access  
- **Example:** Virus, Trojan  
- **Prevention:**  
  - Install antivirus  
  - Avoid downloading from untrusted sources  

---

# Ransomware
- **Definition:** Locks user data and demands payment  
- **Example:** WannaCry attack  
- **Prevention:**  
  - Backup data  
  - Keep system updated  

---

# Social Engineering
- **Definition:** Manipulating users to reveal confidential info  
- **Example:** Fake tech support calls  
- **Prevention:**  
  - Never share OTP/password  
  - Verify identity  

---

# Data Breach
- **Definition:** Unauthorized access to confidential data  
- **Example:** Company database leaks  
- **Prevention:**  
  - Strong passwords  
  - Enable MFA
